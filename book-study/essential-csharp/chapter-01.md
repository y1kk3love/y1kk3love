# 1장 — C# 소개

## 실행 구조

| 구성 요소 | 설명 |
|-----------|------|
| **어셈블리** | CIL(Common Intermediate Language)의 2차 언어로 구성 |
| **컴파일러** | 소스 파일 → 어셈블리어 |
| **VES / Runtime** | 어셈블리어 → 기계어 |

- **관리되는 실행**: 런타임의 제어 아래의 실행 과정 ↔ 네이티브 코드: 런타임 필요 없음
- **관리되는 코드**: 런타임과 같이 에이전트의 컨텍스트 아래에서 실행되는 코드

---

## 런타임 장점

| 장점 | 설명 |
|------|------|
| 언어 상호 운용성 | 사용하는 언어가 달라도 같은 어셈블리어로 변환하기 때문에 다른 원본 언어 사이의 상호 운용성에서 장점을 가짐 |
| 형식 안전성 | 형식 간의 변환을 검사하고, 호환 형식 간의 변환만 보장. [버퍼 오버런](./terms.md) 발생을 방지 |
| 코드 액세스 보안 | 어셈블리 개발자의 코드가 컴퓨터에서 실행할 권한을 지녔는지 인증 |
| 가비지 컬렉션 | 이전에 런타임에서 할당된 메모리를 자동으로 해제하는 메모리 관리 기능 |
| 플랫폼 이식성 | 한 플랫폼에 의존적인 라이브러리를 사용하지 않는 이상 잠재적으로 다양한 운영체제에서 동일한 어셈블리를 실행하도록 지원 |
| BCL (Base Class Library) | 개발자가 자체적인 코드를 개발하지 않아도 되도록 대규모 코드 기반을 제공 |

---

## ILDASM

어셈블리를 분석하고 컴파일러가 생성한 CIL을 텍스트로 추출한다.  
역컴파일화한 결과는 기계어에 비해 이해하기 쉽다.  
난독화 도구 등을 이용하거나 액세스하지 못하게 하여 디스어셈블 작업을 방지할 수 있다.
