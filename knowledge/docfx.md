## 1. DocFX란?

**DocFX**는 Microsoft에서 만든 오픈 소스 **정적 사이트 생성기(Static Site Generator)**입니다.  
C# 소스 코드 내의 XML 주석을 읽어서 자동으로 API 레퍼런스 문서를 만들어주고, 마크다운으로 작성한 일반 문서와 결합하여 하나의 기술 문서 사이트를 구축해주는 도구입니다.

---

## 2. 주요 장점 (Pros)

- **자동 API 문서화:** 소스 코드(`///`) 주석을 기반으로 클래스, 메서드, 프로퍼티 설명을 자동으로 생성합니다.
- **마크다운 통합:** 단순 API 목록뿐만 아니라, 사용 설명서나 튜토리얼을 마크다운으로 작성하여 사이트에 함께 포함할 수 있습니다.
- **상호 참조(Cross Reference):** 코드 내의 특정 클래스 이름을 언급하면 자동으로 해당 API 페이지로 연결되는 링크를 생성합니다.
- **커스터마이징:** HTML/CSS 템플릿을 수정하여 프로젝트만의 고유한 문서 디자인을 적용할 수 있습니다.
- **다양한 출력 포맷:** 정적 웹사이트(HTML)뿐만 아니라 PDF 파일로도 문서를 뽑아낼 수 있습니다.

---

## 3. 주요 단점 및 주의점 (Cons)

- **초기 설정의 복잡함:** `docfx.json` 설정 파일을 직접 만져야 하는데, 처음 접하면 옵션이 많아 다소 어렵게 느껴질 수 있습니다.
- **학습 곡선:** 커스텀 템플릿을 만들려면 Mustache나 Jint 같은 템플릿 엔진에 대한 이해가 추가로 필요합니다.
- **빌드 시간:** 프로젝트 규모가 매우 클 경우, 문서 생성 시간이 길어질 수 있습니다.

---

## 4. 핵심 활용 사례

1. **오픈 소스 라이브러리:** 외부 개발자들을 위한 API 명세서 제공.
2. **사내 기술 블로그/위키:** 프로젝트의 히스토리와 API 문서를 한곳에서 관리.
3. **유니티 프로젝트 문서화:** 복잡한 유니티 스크립트 구조를 체계적으로 정리.

---

## 5. 예제 코드 및 사용법

### 설치 (dotnet tool)

```bash
# 글로벌 도구로 DocFX 설치
dotnet tool install -g docfx
```

### 프로젝트 초기화 및 빌드

```bash
# 기본 템플릿으로 문서 프로젝트 생성
docfx init

# 문서 생성 및 로컬 서버 실행 (http://localhost:8080)
docfx --serve
```
